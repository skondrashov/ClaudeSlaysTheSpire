**Turn 1 (Offensive Mode):** Eruption+ (9 dmg, enter Wrath) -> Consecrate (10 in Wrath) -> Strike (12 in Wrath) = 31 dmg, triggers Mode Shift to Defensive. Then SF+ setup + end turn retaining Miracle/Tranquility. **Correct execution.**

**Turn 3 (ATTACK_BUFF intent): THE FATAL ERROR.**
Hand: Miracle, Consecrate, Strike, Eruption+, Strike, Defend.
Player correctly identified: NO Wrath exits in hand (Tranquility exhausted, Vigilance in draw pile). Eruption+ too dangerous without exit.
Plan: `play 1; play 2 0; play 3 0; play 4; end` = Consecrate, Strike, Strike, Defend.
**What actually happened:** The command `play 1; play 2 0; play 3 0; play 4; end` was a batched `turn()` sequence. After playing Miracle (index 1 was actually supposed to be Consecrate, but Miracle was played first), the card indices shifted. The sequence then played Consecrate -> Strike -> **Eruption+** (at the shifted index 3, which was supposed to be the second Strike) -> attempted to play Defend.

The translated output confirms: `"Consecrate -> Strike -> The Guardian -> Eruption+ -> The Guardian -> Strike -> End Turn"` -- Eruption+ was played instead of the second Strike.

**Result:** Entered Wrath with no exit. Guardian attacks for 32 (16 doubled in Wrath). With 6 block and 38 HP (after Miracle + Consecrate Defend), took 26 damage. HP: 44 -> 12.

**Turn 4:** Guardian attacks 10x4 = 40. At 12 HP with max block 15 (Vigilance 9 + Defend 6), took 25 lethal damage. Dead.

**Turn 5-6:** The log shows additional turns being planned and played, but the player was already in a death spiral from the Turn 3 misplay. At 7 HP with no way to survive 32+ incoming, the run was over.

**Root cause:** Card index shifting in a batched `turn()` command. Playing card at index 1 removed it from hand, shifting all subsequent indices. The sequence `play 1; play 2 0; play 3 0; play 4` assumed static indices, but after each play, the remaining cards shifted down. This caused Eruption+ (which was at index 3 in the original hand) to be played when the player intended to play a Strike.

## Recurring Patterns

1. **Batched turn() index shifting is the #1 execution risk.** This is the second time (the player's own analysis references it) that card indices in a multi-card `turn()` command have caused the wrong card to be played. The combat heuristics explicitly warn: "Hand indices shift after each play. Before entering `play N`, confirm the card at index N matches your plan." The batched `turn()` command violates this rule by pre-computing all indices before any cards are played.

2. **Shop/UI index mismatches.** Multiple instances of `choose N` translating to an unexpected item. In the first run attempt: `choose 5` = "Buy Dramatic Entrance" when intending Rushdown. In the second run: `choose 0` = "Buy Follow-Up" when intending card removal. The underlying purchases may have been correct (hard to verify without seeing the actual shop layout), but the translation layer is unreliable.

3. **Thin deck vulnerability to bad draws.** The 12-card deck was excellent for consistency but left the player vulnerable when key cards (Eruption+, Consecrate, SF+) didn't appear in the same hand. The F14 slime fight (23 HP loss) was caused entirely by drawing Strike + 4 Defend + Miracle against 5 enemies with no Wrath entry.

## Flags for Curator

1. **CRITICAL: `turn()` batched commands must be prohibited or redesigned.** The index-shifting bug in batched play commands is a systemic execution failure, not a player error. The combat heuristics already say "re-read hand state after each card play" but `turn()` sequences bypass this. **Recommendation:** Add an explicit heuristic rule: "NEVER batch multiple targeted plays in a single `turn()` command. Play each card individually with a state check between plays." Or: "In `turn()` sequences, use card NAMES not indices."

2. **Shop index mapping needs investigation.** The translated shop items don't match player intent. Either the shop indices are offset from what the player sees, or the translation layer is buggy. This hasn't caused a gameplay-impacting error yet (removals went through correctly), but it will eventually cause a wrong purchase.

3. **Guardian heuristic should be documented.** The Guardian's Mode Shift (30 damage threshold) + Sharp Hide + Offensive/Defensive cycling is a complex boss pattern. The player handled it well strategically but the heuristic file for Guardian should include specific guidance on: (a) never entering Wrath without a guaranteed exit during Sharp Hide phase, (b) damage budgeting per Mode Shift cycle, (c) the doubled Sharp Hide damage in Wrath (6 per attack vs 3 normally).
