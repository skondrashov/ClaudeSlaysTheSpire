# Rest Site Decisions

- **Upgrade vs Rest Framework**: Run the live-priced ledger (see [[layer:heuristics, hp-management]]): derive what each fight between this rest site and the next recovery will cost THIS deck, including the elite or boss if one is coming. **Rest when current HP doesn't cover that sum with margin for a spike; otherwise upgrade.** An upgrade compounds for the rest of the run; resting buys back roughly one fight's worth of HP — so when the ledger clears, the upgrade is almost always the better purchase.
- **The Upgrade Death Spiral**: Zero upgrades = death sentence. Unupgraded cards → longer fights → more HP loss → forced resting → no upgrades. Default: upgrade whenever the ledger clears. [[cards/Bash]] should be upgraded by Floor 8.

[[cards/Thunderclap]] (7 vs 4 AOE). 9. SIO (11 vs 8). 10. [[cards/Headbutt]] (12 vs 9). 11. Defend (8 vs 5). Context overrides: Corruption above Bash if it's the engine piece.
