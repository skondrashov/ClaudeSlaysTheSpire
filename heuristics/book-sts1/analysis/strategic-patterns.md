# Strategic Patterns

Decisions that were technically correct (math right) but strategically wrong (plan bad). Priority 2 — after [[layer:heuristics, analysis/prediction-errors]].

Examples:
- Correct damage calculation, but attacked when blocking was better
- Correctly identified card values, but picked the wrong one for the deck's needs
- Correct map reading, but chose a dangerous path when a safe one existed
- Correct boss mechanics, but used resources too aggressively early in the fight
